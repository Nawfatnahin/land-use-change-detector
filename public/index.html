How to use

  1. Use the polygon tool in the top-left of the map to draw your region of interest.
  2. Adjust the sensor, time period, and acceptable cloud cover.
  3. Select an index (like NDVI for vegetation) and a change threshold.
  4. Click Run Analysis to fetch tiles directly from Google Earth Engine.

Why use this

Understanding environmental shifts at scale is critical for sustainability. Use this tool to track deforestation in the Amazon, monitor urban sprawl in rapidly developing cities, or observe the shrinking of lakes due to severe drought.

Data Sources

  • Google Earth Engine: Cloud computing platform for planetary-scale geospatial analysis.
  • Sentinel-2: Copernicus program high-resolution multispectral imaging mission (10m).
  • Landsat 8/9: USGS / NASA Earth observation satellite program (30m).

Parameter Guide

Sensor
Sentinel-2 (10 m) offers high spatial detail. Landsat 8/9 (30 m) spans a longer historical archive, dating back to 1972.
Time Period
Select two dates to compare. A greater temporal gap will surface more significant land-cover transitions.
Cloud Cover
Maximum allowable cloud fraction per scene. A lower value yields cleaner composites but may reduce scene availability in cloudy regions.
Index
NDVI — vegetation density. NDBI — built-up surfaces. MNDWI / NDWI — open water. EVI — enhanced vegetation with soil correction.
Threshold
Minimum index delta required to classify a pixel as “changed.” A higher value admits only substantial transitions, suppressing noise.
